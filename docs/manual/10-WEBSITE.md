# 10. WebSite

대표 웹사이트, 콘텐츠, 발행 흐름과 공개 상태를 조직의 외부 표현 관점으로 관리합니다.

## 이 Perspective가 답하는 질문

- 어떤 사이트와 콘텐츠가 현재 공개돼 있는가?
- 발행 버전, 도메인과 배포 상태가 일치하는가?
- 장애나 롤백이 브랜드와 고객 경험에 미치는 영향은 무엇인가?

## 주요 대상

- 대표 사이트와 콘텐츠 채널
- CMS, asset와 발행 workflow
- domain, Route, CDN과 cache
- release, preview, rollback과 공개 상태

## 운영 시작

콘텐츠 작성과 기술 배포를 분리하되 하나의 release 증거로 연결합니다. Preview에서 승인된 결과만 공개하고, 장애 시 마지막 검증 버전으로 되돌릴 수 있어야 합니다.
