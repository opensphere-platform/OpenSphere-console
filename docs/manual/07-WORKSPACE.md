# 7. Workspace

사내 업무 서비스, 프로젝트와 협업 자원을 직원의 일하는 공간으로 구성합니다.

## 이 Perspective가 답하는 질문

- 사용자가 자신의 업무 도구와 프로젝트를 어디서 찾는가?
- 내부 서비스의 접근 권한과 상태는 정상인가?
- 여러 시스템의 알림과 작업을 어떻게 한 흐름으로 연결하는가?

## 주요 대상

- 프로젝트, Namespace와 업무 환경
- 메일, 문서, 승인, 일정과 협업 서비스
- 내부 포털, 자동화와 legacy workload
- 사용자별 즐겨찾기, 최근 작업과 알림

## 운영 시작

Workspace는 서비스 소유권과 사용자 권한을 결합해 보여줍니다. 서비스 자체의 관리 기능은 해당 domain에 남기고, Main Shell은 탐색·인증·공통 알림과 감사만 소유합니다.
