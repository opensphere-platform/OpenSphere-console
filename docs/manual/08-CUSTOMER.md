# 8. Customer

고객, 구독, 고객 포털과 고객별 워크로드를 서비스 결과 중심으로 관리합니다.

## 이 Perspective가 답하는 질문

- 고객이 어떤 서비스와 환경을 사용하고 있는가?
- 구독, 권한, SLA와 실제 서비스 상태가 일치하는가?
- 고객 이슈의 영향 범위와 책임 소유자는 누구인가?

## 주요 대상

- Customer realm과 CIAM
- 고객 포털, tenant와 구독
- 고객 workload, 정책과 네트워크 경계
- SLA, 지원 요청과 서비스 상태

## 운영 시작

고객 신원과 Console 관리 신원을 분리하고 명시적인 federation 계약으로 연결합니다. 고객별 데이터와 작업은 tenant 범위를 벗어나지 않아야 하며 감사 기록은 장기 보존됩니다.
