# Extensions 관리 작업 재평가

## 범위

- 화면: `https://localhost:8090/manage/extensions`
- 작업: `관리 작업` 아코디언의 기본 상태, 펼침 상태, Catalog 탭과의 결합
- 캡처일: 2026-07-22 KST

## 판정

세 기능을 하나의 상위 범주인 `관리 작업`으로 묶는 개념은 타당하다. 그러나 세 개의 폼을 하나의 인라인 아코디언 안에 동시에 배치한 현재 표현 방식은 부적절하다.

세 기능의 성격이 다르기 때문이다.

| 기능 | 빈도 | 위험 | 권장 진입점 |
|---|---|---|---|
| OCI 이미지 설치 | 상대적으로 높음 | 중간, 검증 후 설치 | Catalog 탭의 주 작업 버튼 |
| Private Registry 자격증명 | 낮음 | 높음, 비밀정보 변경 | `관리 작업` 메뉴의 별도 설정 Drawer/Modal |
| Digest 철회 | 매우 낮음·긴급 | 매우 높음, append-only/비가역 | 공급망 보안 또는 Audit 문맥의 Danger flow |

## 근거

1. [기본 상태](01-management-actions-collapsed.png): 화면은 정돈되지만 핵심 설치 동작이 일반적인 `관리 작업` 아래 숨는다.
2. [펼침 상태](02-management-actions-expanded.png): 서로 다른 세 폼이 같은 카드 위계로 동시에 나타나며, 기존 Topology 문맥을 아래로 밀어낸다.
3. [Catalog 결합](03-catalog-with-management-actions.png): 아코디언이 탭 전환 후에도 유지되어 Catalog 목록보다 관리 폼이 먼저 보인다. 현재 탭의 작업 문맥과 분리되지 않는다.

## 강점

- 접힌 기본 상태에서 화면 밀도를 낮춘다.
- 아코디언에 `aria-expanded`와 이름이 있는 region이 연결되고, 내부 섹션은 `h2`와 입력 레이블을 가진다.
- Registry 설정 상태와 철회 건수를 노출한다.
- 설치가 `검증 → 설치` 2단계이고 필수 입력 전 버튼이 비활성화된다.

## 주요 위험

1. **P1 — 비가역 동작의 보호가 부족하다.** Digest 철회와 Registry 자격증명 제거가 일반 폼 버튼으로 실행되며, 코드상 명시적 확인 단계나 영향 범위 미리보기가 없다.
2. **P1 — 핵심 설치 흐름의 발견성이 낮다.** 확장 관리의 대표 동작인 설치가 접힌 일반 범주 안에 숨는다.
3. **P2 — 세 작업의 위험도와 사용 빈도가 시각적으로 평준화된다.** 자격증명 설정, 설치, 긴급 철회가 같은 표면과 간격을 사용한다.
4. **P2 — 탭 문맥을 침범한다.** 펼친 패널이 Topology/Catalog/Audit 등 모든 탭 위에 남아 현재 콘텐츠를 아래로 밀어낸다.
5. **P2 — 긴 키보드 이동과 비활성 사유 전달 문제가 있다.** 하나의 region 안에 많은 입력이 연속되고, 길이 조건이 주로 placeholder에 의존해 비활성 버튼의 이유를 즉시 이해하기 어렵다.
6. **P3 — 한영 혼용이 작업 스캔을 늦춘다.** `Private GHCR access`, `OCI image revocation ledger`, `OCI 이미지로 설치`가 동일 수준에서 서로 다른 언어 규칙을 쓴다.

## 권장 구조

- 상단 또는 Catalog 툴바: 기본 버튼 `확장 설치`
- 보조 버튼: `관리 작업 ▾`
  - `Private Registry 설정`
  - `Digest 철회`
- 각 명령은 별도 Drawer/Modal로 열어 현재 탭을 밀어내지 않는다.
- Digest 철회는 Danger flow로 분리한다.
  - 영향받는 활성/설치 대기 패키지 미리보기
  - exact digest 재확인
  - 승인 사유
  - 비가역성 안내
  - 최종 확인 단계
- Registry 제거도 별도 확인과 영향 범위를 제공한다.

즉, `관리 작업`이라는 상위 그룹은 유지하되 `세 폼 동시 펼침`은 제거하는 것이 가장 합리적이다.

## 증거 한계

스크린샷과 DOM, 관련 템플릿/이벤트 처리 코드를 확인했다. 실제 스크린리더 낭독 순서, 전 구간 키보드 이동, 200~400% 확대 reflow는 별도 시험이 필요하다. 비가역 동작은 평가 과정에서 실행하지 않았다.
